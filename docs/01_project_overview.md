# Hospital Management System (HMS)

## Project Overview

The Hospital Management System (HMS) is a modern full-stack web application designed to digitize and simplify hospital operations. It provides a centralized platform for managing patients, doctors, appointments, prescriptions, laboratory reports, pharmacy inventory, billing, and administrative tasks.

The system improves communication between different hospital departments, reduces paperwork, minimizes manual errors, and enables secure access to medical records through role-based authentication.

This project follows industry-standard software architecture with separate frontend and backend applications connected through REST APIs and a MongoDB database.

---

## Problem Statement

Many hospitals still rely on manual paperwork or disconnected software systems to manage patient records, appointments, prescriptions, billing, and laboratory reports. These traditional methods often result in delays, duplicate records, data inconsistency, and inefficient communication between departments.

There is a need for a centralized, secure, and scalable web-based Hospital Management System that streamlines daily operations and improves the overall patient experience.

---

## Project Objectives

- Digitize hospital management processes.
- Maintain centralized patient records.
- Enable online appointment scheduling.
- Manage doctors, staff, and departments efficiently.
- Generate digital prescriptions and bills.
- Store laboratory reports securely.
- Manage pharmacy inventory.
- Implement secure authentication and role-based access control.
- Generate reports and analytics for hospital administrators.
- Deploy the application using modern cloud technologies.
